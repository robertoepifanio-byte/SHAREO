Escopo & Metodologia

Auditoria do ShareO ● marketplace de economia circular para aluguel local de itens, sob o framework de 6 aspectos (Performance, Usabilidade, Design, Funcionalidades, Segurança, Estratégia). Diferente de uma avaliação superficial de site, cada afirmação foi verificada contra o código-fonte: next.config.ts, middleware.ts, relatório Lighthouse real, resultado k6, varredura de next/image e SEO. O objetivo é separar riscos genéricos de template dos riscos reais deste produto.

1 — Performance & Infraestrutura
⚡
Vercel + CDN global · Next.js 15 App Router (SSR/ISR)
A
Confirmado no código
  • Lighthouse real 97 · LCP 2.3s · CLS 0 · TBT 70ms · FCP 1.3s
  • k6: 50 VUs, P95=159ms, 0% falhas (4117 iterações)
  • Region fixada gru1 (São Paulo) — latência 71× menor vs iad1
  • next/image com formats: avif, webp + resize on-the-fly (17 componentes)
  • Cache headers + BF Cache nas páginas institucionais
Riscos reais
  • public/icones/ (PT) ≈ 13 MB de PNGs 2 MB — zero referências: peso morto no repo
  • PWA icons 192/512 com 944 KB cada (esperado ~30–50 KB)
  • 8 <img> cru (avatar/foto) fora do next/image em 7 arquivos
Veredito: "imagens pesadas" do template — mitigado na entrega (next/image resize); ação real é faxina de repo e PWA icons, não experiência do usuário. "SEO fraco por SPA" não se aplica: o app é SSR/ISR.
2 — Usabilidade & Experiência (UX)

Mobile-first · WCAG 2.1 AA · navegação clara

4 — Funcionalidades & Integrações
🧩
207 funcionalidades · 52 endpoints · APIs com fallback
A
Confirmado no código
  • Ciclo completo: buscar → reservar → pagar → retirar → devolver → avaliar
  • Validação Zod server-side em todos os endpoints
  • Integrações isoladas: Stripe (webhook), Supabase Realtime, ViaCEP, Mapbox, Resend
  • Fallbacks: filtro de distância sem GPS usa cidade do perfil; CSP com allowlist de domínios
  • Modularidade: lib/pricing, lib/platform-config, lib/email desacoplados
Riscos reais
  • NextAuth v5 ainda em beta — monitorar breaking changes pré-GA
  • Repasse PIX é manual no MVP (automação bancária só em V1+)
  • Stripe em Test mode — Live mode bloqueado por D4
Veredito: risco "falhas de API sem fallback" do template é parcialmente mitigado — há fallback em geolocalização e allowlist CSP; recomenda-se circuit-breaker explícito para Mapbox/Resend em produção.
5 — Segurança
🔒
6 headers · CSP com nonce · guards server-side
A
Confirmado no código
  • Headers: X-Frame-Options, nosniff, Referrer-Policy, Permissions-Policy, HSTS preload
  • CSP gerado por request com nonce no middleware.ts
  • IDOR: guard ownerId !== session.user.id → 403 em todas as rotas de recurso
  • CPF/CNPJ criptografados em repouso (ADR-005) · zero PII em logs/Sentry
  • Rate limiting (Upstash) · CSRF nativo NextAuth · JWT 15min + refresh rotativo 7d
  • 37 testes E2E de segurança (XSS, IDOR, anti-enumeração, rate limit)
Riscos reais
  • RLS desabilitado (incompatível com PgBouncer) — segurança depende 100% dos guards
  • Pentest externo ainda não realizado — planejado no roadmap
  • PCI-DSS formal só em H3 (cartão nunca trafega nos servidores ShareO)
Veredito: risco "vulnerabilidades em formulários" do template não se aplica — toda entrada passa por Zod + sanitização + CSP. Como RLS está off por design, manter cobertura de teste dos guards é crítico.

Achados Acionáveis (sem bloqueador externo)
P1
Remover public/icones/ não referenciado (~13 MB)
Seis PNGs de ~2 MB (eletronicos.png, Moda.png, etc.) + arquivos temp (composite-temp.png.png) com zero referências no código. Os ícones realmente usados estão em /icons/ (EN). Deletar ou mover para fora do repo + .gitignore. Reduz clone/deploy sem impacto em produção.
P2
Comprimir PWA icons 192/512 (944 KB → ~40 KB)
São carregados pelo browser direto do manifest.ts (fora do next/image). Reprocessar com sharp para PNG otimizado/quantizado. Melhora tempo de instalação do PWA e Lighthouse PWA.
P2
Migrar 8 <img> de conteúdo do usuário para next/image
Avatares e fotos de avaliação em perfil/, reservas/, _ReviewForm.tsx, UserDropdown.tsx. O remotePattern do Supabase já está configurado — habilita avif/webp + lazy + dimensões fixas (reduz CLS futuro).
P3
Circuit-breaker / fallback explícito para Mapbox e Resend
Hoje há fallback em geolocalização e allowlist CSP, mas falha de tile Mapbox ou de envio Resend não tem degradação graciosa documentada. Adicionar timeout + estado de erro amigável e retry no envio de e-mail crítico.
P3
Rodar Lighthouse pós-gru1 e arquivar baseline mobile
O relatório atual (97) é de 06/06, antes do pin gru1. Rodar de novo em perfil mobile + throttling 4G e versionar como baseline para o gate de CI comparar regressões.
