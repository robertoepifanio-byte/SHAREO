Usuário & Acesso
🔐 Autenticação 10 H1
  • Login e-mail + senha (NextAuth.js v5)
  • Cadastro PF com CPF e validação
  • Recuperação de senha via token por e-mail
  • Redefinição de senha
  • Logout com redirect
  • Sessão JWT com refresh token rotativo (7d)
  • Middleware: proteção de 8 rotas privadas
  • Auth mobile — endpoints separados (JWT)
  • Rate limiting em endpoints de auth
  • Redirecionamento pós-login com callbackUrl
👤 Perfil 9 H1
  • Edição: nome, bio, telefone, cidade, estado, bairro, avatar
  • Perfil público por ID (/perfil/[id])
  • Verificação de identidade (documentos — CPF/CNPJ)
  • Selo "Verificado" pós-aprovação
  • Estatísticas: itens, aluguéis, nota média
  • Avaliações recebidas (últimas 5 de N)
  • Data de membro desde
  • Tipo de conta: PF ou PJ
  • Upgrade de PF → PJ
⚖️ LGPD 7 H1
  • Exportar dados pessoais em JSON (art. 20 LGPD)
  • Excluir conta com confirmação (soft-delete)
  • Verificação de identidade — CPF/CNPJ criptografados
  • Status de verificação: pendente / aprovado / rejeitado + motivo
  • Política de Privacidade (/privacidade)
  • Termos de Serviço (/termos)
  • Zero PII em logs, zero PII em respostas de API desnecessárias
Itens & Busca
🔍 Busca & Listagem 13 H1
  • Listagem paginada de itens (/itens) — SSR
  • Busca por palavra-chave (título)
  • Filtro por categoria (7 categorias com ícones SVG)
  • Filtro por cidade / estado
  • Filtro por faixa de preço
  • Filtro por condição (novo, seminovo, bom estado, regular)
  • Filtro por nota mínima (minRating)
  • Filtro por distância (geolocalização)
  • Chips de filtros ativos com remoção individual
  • Ordenação: mais recentes, menor preço, melhor avaliação
  • Toggle visualização em mapa (Mapbox / Leaflet)
  • Pull to refresh (mobile)
  • Skeleton loading + estado vazio com CTA
📄 Detalhe do Item 10 H1
  • Galeria de fotos com navegação
  • Informações completas: título, descrição, categoria, condição, localização
  • Calculadora de locação (diário / semanal / mensal) com total automático
  • Calendário de disponibilidade
  • Preço de compra vs. aluguel — economia estimada ao locatário
  • Card do proprietário com nota e selo de verificado
  • Botão favoritar / desfavoritar inline
  • CTA sticky de solicitação de reserva
  • Contador de visualizações (viewCount)
  • Link de compartilhamento público (sem autenticação)
📦 Gestão de Anúncios 11 H1
  • Criar anúncio: título, descrição, categoria, fotos, preço, condição, localização, caução
  • Upload de imagens para Supabase Storage
  • Preview do anúncio antes de publicar
  • Indicador de qualidade do anúncio (ListingQualityIndicator)
  • Sugestão de preço baseada em itens similares
  • Editar anúncio existente
  • Ativar / desativar anúncio
  • Excluir anúncio (soft delete)
  • Listagem "Meus Anúncios" com status de aprovação
  • Itens aguardando aprovação invisíveis no feed público
  • Cards com ações inline: editar, ativar, excluir
❤️ Favoritos 3 H1
  • Favoritar item (card e página de detalhe)
  • Desfavoritar item
  • Lista de itens favoritados (/favoritos) com acesso rápido
Ciclo de Locação
📅 Reservas 20 H1
  • Solicitar reserva com nota ao proprietário
  • 7 estados: PENDING → CONFIRMED → ACTIVE → RETURNED → COMPLETED | CANCELLED | DISPUTED
  • Barra de progresso visual do status da reserva
  • Confirmar reserva (proprietário)
  • Cancelar reserva com motivo (ambos — grátis até 24h antes)
  • Pagar via Stripe (locatário após confirmação)
  • Pagamento retido até confirmação de entrega
  • Taxa ShareO 10% explicitada no resumo financeiro
  • Caução cobrada no checkout e devolvida após devolução sem danos
  • Marcar como Ativo — proprietário confirma entrega
  • Countdown de devolução em tempo real
  • Checklist de devolução (locatário)
  • Formulário de condição pós-devolução (proprietário)
  • Taxa de atraso automática (lateFeeAmount)
  • Extensão de prazo de locação
  • Fotos de check-in e check-out (upload pelo proprietário)
  • Contrato digital — visualizar e assinar (ContractBanner)
  • Polling de status sem refresh manual (BookingStatusPoller)
  • Link direto para chat a partir da reserva
  • Crons automáticos: auto-cancelamento, expiração, lembretes, devolução
💬 Chat 6 H1
  • Chat em tempo real via Supabase Realtime

  • Editar avaliação já enviada
  • Nota média exibida no perfil e nos cards de item
💳 Pagamentos 7 H1

  • Webhooks Stripe para atualização de status em tempo real
  • Dados de cartão nunca trafegam nos servidores ShareO
  • Resumo financeiro: diárias × preço, taxa 10%, caução, total
  • Status de pagamento: PENDING | PAID
  • Retorno visual pós-pagamento (?payment=success / ?payment=cancelled)
  • Disputas com análise manual pela equipe ShareO

Qualidade & Conformidade
♿ Acessibilidade 9 H1
  • Contraste mínimo 4.5:1 — WCAG 1.4.3 AA em todos os textos
  • Landmarks semânticos: <main>, <nav>, <header>, <footer>
  • ARIA roles: alert, tab, tablist, tabpanel
  • Navegação completa por teclado (Tab, Enter, Espaço)
  • focus-visible rings em todos os elementos interativos
  • alt descritivo em todas as imagens
  • aria-hidden em ícones decorativos
  • Tap targets ≥ 44×44px em mobile (WCAG 2.5.5)
  • Labels associados a todos os inputs de formulário
🔒 Segurança 8 H1
  • Headers HTTP: CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy, HSTS
  • XSS: sanitização de inputs + CSP bloqueando scripts inline não autorizados
  • IDOR: Row Level Security (RLS) no Supabase — 75 políticas
  • Validação Zod em todos os endpoints de API (server-side)
  • CPF / CNPJ criptografados em repouso (ADR-005)
  • Zero PII em logs / Sentry
  • Rate limiting em auth e endpoints sensíveis
  • CSRF protegido via NextAuth (tokens CSRF nativos)
📱 Usabilidade & UX 12 H1
  • Design mobile-first: base 375px → 768px → 1280px
  • Navegação inferior (BottomNav) em mobile
  • Menu lateral mobile (hamburguer)
  • Botão de logout no header desktop
  • Loading skeletons em cards e listas
  • Estados vazios com mensagem contextual e CTA
  • Error boundaries com recovery automático
  • Toast notifications (Sonner) para feedback de ações
  • Pull to refresh em listas mobile
  • Cards com swipe (SwipeableItemCard)
  • Formulários com validação inline em tempo real
  • Estados de erro em campos com contraste WCAG AA
🔎 SEO 6 H1
  • Metadata API do Next.js: title 60 chars, description 155 chars, canonical, og:image
  • Sitemap dinâmico gerado pelo Next.js
  • robots.txt com User-Agent e Disallow configurados
  • URLs semânticas: /itens/[id], /loja/[slug]
  • ISR para páginas de itens populares (revalidação automática)
  • JSON-LD estruturado — Product, Offer, LocalBusiness (H2)
📲 PWA & Offline 3 H1
